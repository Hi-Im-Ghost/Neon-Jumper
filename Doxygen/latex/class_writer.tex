\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename Output\+Stream, typename Source\+Encoding = UTF8$<$$>$, typename Target\+Encoding = UTF8$<$$>$, typename Stack\+Allocator = Crt\+Allocator, unsigned write\+Flags = k\+Write\+Default\+Flags$>$\newline
class Writer$<$ Output\+Stream, Source\+Encoding, Target\+Encoding, Stack\+Allocator, write\+Flags $>$}
JSON writer. 

\mbox{\hyperlink{class_writer}{Writer}} implements the concept Handler. It generates JSON text by events to an output os.

User may programmatically calls the functions of a writer to generate JSON text.

On the other side, a writer can also be passed to objects that generates events,

for example \mbox{\hyperlink{class_generic_reader_a0c450620d14ff1824e58bb7bd9b42099}{Reader\+::\+Parse()}} and Document\+::\+Accept().

Sets the maximum number of decimal places for double output. 

This setting truncates the output with specified number of decimal places.

For example,


\begin{DoxyCode}{0}
\DoxyCodeLine{writer.SetMaxDecimalPlaces(3);}
\DoxyCodeLine{writer.StartArray();}
\DoxyCodeLine{writer.Double(0.12345);                 \textcolor{comment}{// "{}0.123"{}}}
\DoxyCodeLine{writer.Double(0.0001);                  \textcolor{comment}{// "{}0.0"{}}}
\DoxyCodeLine{writer.Double(1.234567890123456e30);    \textcolor{comment}{// "{}1.234567890123456e30"{} (do not truncate significand for positive exponent)}}
\DoxyCodeLine{writer.Double(1.23e-\/4);                 \textcolor{comment}{// "{}0.0"{}                  (do truncate significand for negative exponent)}}
\DoxyCodeLine{writer.EndArray();}

\end{DoxyCode}


The default setting does not truncate any decimal places. You can restore to this setting by calling
